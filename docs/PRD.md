# DevToolkit - 开发者轻量工具箱

## 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品定位

**DevToolkit** 是一款面向开发者的轻量级桌面工具箱，专注于解决日常开发中高频但繁琐的小问题。

**核心理念**：小而美、开箱即用、无需联网、可扩展。

### 1.2 目标用户

- 后端/全栈开发者
- DevOps 工程师
- 需要快速诊断本地环境的运维人员
- 技术管理者（快速了解团队成员的本地环境状态）

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 快 | 启动即用，无需配置 |
| 轻 | 单文件运行，占用资源极低 |
| 准 | 输出精确，无需解读 |
| 扩展 | 插件化架构，按需添加工具 |

## 2. 痛点分析

### 2.1 端口管理痛点

| 痛点 | 现状 | 期望 |
|------|------|------|
| 端口占用排查 | 需要手动敲命令查进程 | 一键查看当前所有端口占用 |
| 释放端口 | 需要记住进程ID，手动kill | 一键强制释放指定端口 |
| 端口冲突 | 服务启动失败才知道端口被占 | 启动前预检端口状态 |

### 2.2 用户场景

```
场景一：小明开发时启动了多个服务，换了个项目后端口冲突
  -> 需要快速查看哪些端口被占用，哪些可以复用

场景二：小红启动后端服务时提示端口被占用
  -> 需要快速定位是哪个进程占用了该端口

场景三：小李调试时需要释放某个端口重新测试
  -> 需要一键强制结束该端口对应的进程
```

## 3. 功能规格

### 3.1 核心功能

#### 3.1.1 端口占用查看

**功能描述**：展示当前系统所有被占用的端口信息。

**输出字段**：
- 协议类型（TCP/UDP）
- 端口号
- 进程名称
- 进程ID (PID)
- 连接状态

**交互设计**：
- 启动默认展示端口列表
- 支持按端口号/进程名过滤搜索
- 支持按协议类型筛选（TCP/UDP/ALL）
- 点击行可复制端口号或PID

#### 3.1.2 端口占用清除

**功能描述**：强制结束占用指定端口的进程。

**交互流程**：
1. 输入端口号（或从列表选择）
2. 显示占用该端口的进程信息
3. 用户确认后执行kill
4. 显示执行结果（成功/失败）

**安全设计**：
- 需要用户二次确认，防止误操作
- 显示进程详细信息帮助判断
- 支持只读模式（不执行kill，只诊断）

#### 3.1.3 工具扩展框架

**设计目标**：未来可扩展至10+工具，架构需支持热插拔。

**扩展工具候选**：
- 网络连通性检测
- DNS解析检查
- 本地Hosts文件管理
- 进程管理器
- 日志文件监控
- JSON/Base64编解码

## 4. 技术架构

### 4.1 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 语言 | Rust | 轻量、高性能、跨平台、单二进制 |
| UI框架 | Tauri + React | 轻量、Web技术生态、native体验 |
| 数据获取 | sysinfo crate | 系统信息获取 |
| 构建目标 | Windows / macOS / Linux | 全平台支持 |

### 4.2 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                      DevToolkit                          │
├─────────────────────────────────────────────────────────┤
│                      UI Layer (React)                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ PortView │  │ PortKill │  │ NetCheck │  │  ...   │ │
│  └──────────┘  └──────────┘  └──────────┘  └────────┘ │
├─────────────────────────────────────────────────────────┤
│                    Tool Interface                        │
│                    (Trait: Tool)                         │
├─────────────────────────────────────────────────────────┤
│                    Core Layer (Rust)                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │ PortTool │  │ NetTool  │  │  ...     │  (可扩展)     │
│  └──────────┘  └──────────┘  └──────────┘             │
├─────────────────────────────────────────────────────────┤
│                    Platform Layer                        │
│              (sysinfo / netstat / process)              │
└─────────────────────────────────────────────────────────┘
```

### 4.3 扩展性设计

```rust
// 工具接口定义
trait Tool {
    fn name(&self) -> &str;
    fn description(&self) -> &str;
    fn execute(&self, args: Args) -> Result<String, Error>;
}

// 内置工具注册
struct ToolRegistry {
    tools: Vec<Box<dyn Tool>>,
}

impl ToolRegistry {
    fn register(&mut self, tool: Box<dyn Tool>) { ... }
    fn get(&self, name: &str) -> Option<&Box<dyn Tool>> { ... }
    fn list(&self) -> Vec<&Box<dyn Tool>> { ... }
}
```

### 4.4 跨平台支持

| 平台 | 端口查询方式 | 进程终止方式 |
|------|--------------|--------------|
| Windows | `netstat -ano` | `taskkill /PID xxx /F` |
| macOS | `lsof -i` | `kill -9 xxx` |
| Linux | `ss -tulpn` / `netstat` | `kill -9 xxx` |

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 启动时间 | < 1秒 |
| 端口扫描 | < 2秒 |
| 内存占用 | < 50MB |
| 安装包大小 | < 15MB |

### 6.2 安全要求

- 不上传任何数据，离线运行
- 不修改系统关键配置（除非用户主动操作）
- 进程终止操作需要二次确认

### 6.3 兼容性

- Windows 10/11
- macOS 10.15+
- Ubuntu 20.04+ / Debian 11+
